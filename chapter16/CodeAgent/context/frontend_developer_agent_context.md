# GRIDi BugKiller AI 编程 IDE 网页 UI 设计方案

## 一、整体设计定位

### 核心风格

以「未来科技感 + 高效开发」为核心，采用深色模式为主、高亮色点缀的设计语言，契合编程工具的专业属性，同时通过光影层次、几何线条和动态交互强化科技感，降低长时间编码的视觉疲劳。

### 核心设计原则

1.  **功能优先**：流程节点清晰可视化，操作路径最短化，符合编程用户的高效操作习惯；
2.  **交互流畅**：每个操作都有明确反馈，减少等待焦虑，复杂流程拆解为可感知的步骤；
3.  **视觉聚焦**：关键操作区、状态区高亮处理，次要信息弱化，避免视觉干扰；
4.  **响应式适配**：支持桌面端（优先）、平板端，保证代码输入框、流程导航区在不同尺寸下的可用性。

## 三、配色方案

### 核心配色

-   主色：蓝色（#3B82F6）- 代表科技、专业、信任，用于按钮、选中状态、关键交互元素；
-   辅助色：绿色（#10B981）- 代表成功、完成，用于已执行节点、成功反馈；
-   警告色：黄色（#F59E0B）- 代表等待、提示，用于警告反馈；
-   错误色：红色（#EF4444）- 代表错误、删除，用于重置按钮、错误反馈；
-   中性色：  -   背景色：#111827（页面主背景）、#1F2937（卡片背景）、#2D3748（输入框背景）；
      -   文字色：#F3F4F6（主文字）、#D1D5DB（次要文字）、#9CA3AF（辅助文字）、#6B7280（占位文字）；
      -   边框色：#374151（普通边框）、#4B5563（输入框边框）。

### 配色逻辑

-   深色模式为主：符合编程用户长时间使用的视觉习惯，减少蓝光刺激；
-   高对比度：文字与背景对比度≥4.5:1，确保可读性；
-   渐变应用：仅用于关键元素（选中状态、加载动画、节点序号），增强科技感，避免过度使用导致视觉杂乱。

## 四、交互设计细节

### 1\. 流程导航交互

-   节点跳转：点击流程节点，主内容区平滑滚动至对应节点（滚动时长 300ms），同时节点序号放大闪烁（1.2 倍，持续 500ms）；
-   状态同步：执行完当前节点后，自动标记为 “已执行”，连接线渐变点亮，同时滚动至下一节点（可选关闭自动滚动，在设置中配置）；
-   重置反馈：点击节点重置后，节点状态变为 “待执行”，后续节点同步重置，弹出提示 “节点已重置，可重新操作”（3 秒后消失）。

### 2\. 输入框 / 文本域交互

-   代码输入框：  -   支持语法高亮（Python、Java、JavaScript 等主流语言）；
      -   支持行号显示，滚动时行号同步跟随；
      -   悬浮时右上角显示 “复制”“格式化” 按钮，点击 “复制” 后显示 “代码已复制到剪贴板” 提示（2 秒后消失）；
      -   支持粘贴自动格式化（可选，默认开启）。

-   普通输入框：  -   focus 时边框变色 + 轻微放大（1.02 倍，过渡时长 200ms）；
      -   输入错误时（如格式不符合要求），边框变为红色，下方显示错误提示（12px，#EF4444）。

### 3\. 按钮 / 图标交互

-   功能按钮：  -   hover 时：背景色加深 + 轻微放大（1.05 倍）；
      -   点击时：向下按压效果（缩放 0.98 倍，持续 100ms）；
      -   不可用状态： opacity 0.5，鼠标指针变为 “禁止”，点击无响应。

-   图标按钮：  -   悬浮时显示 tooltip（如 “复制上下文”），背景变为半透明灰色；
      -   操作成功后，图标短暂变色（绿色，持续 1 秒），增强反馈。

### 4\. 特殊节点交互

-   外部执行节点：  -   加载动画同步进度（如 Agent 处理进度，0%-100%，可在设置中关闭）；
      -   支持后台运行，点击 “取消” 可中断当前 Agent 请求，返回上一节点；

-   终端节点：  -   重播图标 hover 时旋转 180°，点击后重置所有节点状态，滚动至第一个节点；
      -   点击 “返回首页”，回到任务类型选择界面，当前任务状态保存（可在 “历史任务” 中查看）。

### 5\. 全局交互

-   快捷键支持：常用操作配置快捷键（如复制 Ctrl+C、生成 Ctrl+G、提交 Ctrl+S），在帮助文档中列出；
-   滚动优化：主内容区滚动时，顶部导航栏和流程导航区固定，避免遮挡；
-   响应式适配：屏幕宽度＜1440px 时，右侧辅助区隐藏，可通过 “展开” 按钮显示（悬浮在左侧操作区右侧）；屏幕宽度＜1024px 时，流程导航区改为下拉选择器，节省空间。

## 五、操作指引设计

### 1\. 节点级指引

-   每个节点的 “操作指引” 文字（流程导航区）简洁明了，仅提示核心操作（如 “粘贴分析结果并保存”）；
-   首次使用某节点时，显示浮动指引气泡（箭头指向核心按钮），内容为详细操作步骤（如 “点击「上下文生成」按钮，系统将自动整合需求信息”），支持 “下次不再显示” 勾选框。

### 2\. 功能提示

-   输入框 placeholder 提示示例内容（如代码输入框 placeholder 显示 “粘贴待优化的 Python 代码，支持 markdown 格式...”）；
-   按钮 hover 时显示 tooltip，补充说明功能（如 “生成” 按钮 tooltip 显示 “生成异常依赖对象源码，需先输入依赖列表”）；
-   帮助入口：顶部导航栏 “帮助文档” 点击后，右侧弹出帮助面板（可折叠），按任务类型和节点分类，提供图文操作指南。

### 3\. 错误引导

-   操作错误时（如未输入内容点击 “提交”），显示红色提示文本，同时核心输入框边框闪烁（红色，3 次，每次 200ms）；
-   Agent 请求失败时（如网络错误），显示错误提示 “请求失败，请检查网络后重试”，并提供 “重新请求” 按钮。

## 六、视觉强化细节

### 1\. 科技感元素

-   背景纹理：页面主背景添加轻微网格纹理（透明度 10%），增强科技感，不影响可读性；
-   光影效果：卡片、输入框添加内阴影（rgba (0,0,0,0.2)），增强层次感；
-   渐变线条：流程导航区已执行节点连接线、按钮 hover 渐变，采用线性渐变（蓝→绿），符合科技产品视觉趋势。

### 2\. 动态效果

-   页面加载：首次进入时，Logo 渐变放大（从 0 到 1 倍，时长 500ms），流程节点依次点亮（从第一个到当前节点，每个间隔 100ms）；
-   节点切换：任务类型切换、节点跳转时，采用淡入淡出 + 轻微位移动画（Y 轴 -10px，透明度从 0 到 1，时长 300ms）；
-   反馈动画：操作成功（如复制、生成）时，显示绿色对勾动画（缩放 1.5 倍后恢复，时长 500ms）；操作失败时，显示红色叉号动画（轻微晃动，时长 300ms）。

## 七、开发适配说明

### 1\. 组件复用建议

-   提取通用组件：流程节点导航组件、代码输入框组件、功能按钮组件、反馈提示组件，确保各任务类型样式统一；
-   表单布局组件：按 “标签 + 输入框 + 按钮” 的结构封装通用表单组件，支持单列、双列、多行布局配置。

### 2\. 兼容性要求

-   浏览器支持：兼容 Chrome（≥90）、Firefox（≥88）、Edge（≥90），不支持 IE；
-   响应式断点：1920px（桌面端）、1440px（小桌面端）、1024px（平板端），低于 1024px 时优化布局为单列。

### 3\. 性能优化建议
